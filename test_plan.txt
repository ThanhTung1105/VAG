MODULE 4: CHECKBOX SUBTASK & APPROVE
═══════════════════════════════════════════════════════

TC-4.1  Tick checkbox subtask (unfinished → finished)
        Bước: Tìm subtask có ☐ → click checkbox
        Kỳ vọng:
          - Checkbox chuyển tím ngay lập tức (không lag)
          - Tên subtask đổi màu tím
          - Cột Trạng thái đổi sang "Chờ duyệt"
          - Cột Tiến độ của Task parent cập nhật (giảm nếu trước đó 100%)
        [ ] Pass / [ ] Fail

TC-4.2  Untick checkbox (finished → unfinished)
        Bước: Click checkbox tím (finished)
        Kỳ vọng:
          - Checkbox trở lại ☐ trống ngay lập tức
          - Tên subtask trở lại màu bình thường
          - Trạng thái → "Chưa hoàn thành"
        [ ] Pass / [ ] Fail

TC-4.3  Approve subtask
        Bước: Click chip "Chờ duyệt" (tím) của 1 subtask
        Kỳ vọng:
          - Hiện popover: "Duyệt" / "Từ chối"
          - Click "Duyệt" → checkbox chuyển xanh lá, trạng thái "Đã duyệt"
          - Cột "Hoàn thành lúc" hiện ngày hiện tại
        [ ] Pass / [ ] Fail

TC-4.4  Reject subtask
        Bước: Click chip "Chờ duyệt" → "Từ chối"
        Kỳ vọng:
          - Subtask quay về "Chưa hoàn thành", checkbox trống
          - "Hoàn thành lúc" trống
        [ ] Pass / [ ] Fail

TC-4.5  Approve tất cả subtask của task
        Bước: Tìm Task có subtask "Chờ duyệt" → click chip trạng thái của TASK
        Kỳ vọng:
          - Popover: "Duyệt tất cả" / "Từ chối tất cả"
          - Click "Duyệt tất cả" → tất cả subtask chờ duyệt → "Đã duyệt"
          - Task status → "Hoàn thành", actual_finish hiện ngày
        [ ] Pass / [ ] Fail

TC-4.6  Actual finish lifecycle
        Bước:
          1. Từ task 100% approved (completed) → thêm 1 subtask mới
          2. Kiểm tra task status + actual_finish
          3. Finish + Approve subtask mới
          4. Kiểm tra lại
        Kỳ vọng:
          Sau bước 1: task status ≠ completed, actual_finish = trống
          Sau bước 3: task status = completed, actual_finish = ngày hôm nay
        [ ] Pass / [ ] Fail
